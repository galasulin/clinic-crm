## 📋 סקירה

מערכת ניהול למרפאת פיזיותרפיה / קליניקה: יומן תורים, תיק מטופל, מנוע **שאלון חכם → אנמנזה קלינית**, CRM לידים עם **משפך מכירות**, הצעות מחיר/הזמנות וחיוב, מלאי וחוזים, שכבת **מוקד טלפוני (CTI)**, פורטל מטופלים לשירות עצמי, ו**פלטפורמת דוחות ואנליטיקה** (דיווח על כל נתון, תזמון, שיתוף, ו"שאל את הנתונים" בשפה חופשית) — עם שכבת **AI** מעל הכל. הרעיון המנחה — **מנועים במקום מסכים**: רוב היכולות *מורכבות* מבונה-טפסים, נתוני-בסיס ומנוע אנמנזה, במקום קוד קשיח. המערכת עובדת ב**זמן-אמת** (יומן חי + התראות דחיפה על WebSockets), כוללת מנוע **אוטומציות ללא-קוד**, וארוזה לרוץ **On-Prem כשירות Windows יחיד** מול שרת SQL ייעודי.

## 🏗️ ארכיטקטורה

![Architecture](assets/architecture.svg)

Clean Architecture בחלוקה ל-**Api / Core / Infrastructure**, React SPA מעל, SQL Server מתחת, וג'ובים ברקע + webhooks חתומים לאינטגרציות יוצאות. האימות הוא JWT עם רוטציית refresh-token; פורטל המטופלים משתמש ב**token נפרד ומוגבל-מטופל** שנדחה על-ידי endpoints של הצוות. ה-Tenancy נאכף לפי `ClinicId`. אינטגרציות חיצוניות (טלפוניה, AI, הודעות, מקורות לידים) מאחורי **מתאמים provider-agnostic**, לכל אחד סימולטור "dev" מובנה כך שהמערכת רצה מקצה-לקצה בלי שום חשבון צד-שלישי.

| **כספים** | מחזור רכש + יתרת **A/P** לספק; דוח **הזדקנות חובות A/R** (0–30 / 31–60 / 61–90 / 90+); חשבוניות נושאות **מספר הקצאה** תואם-רשות-המסים ומופקות ל-**PDF בצד-שרת** |
| **מלאי** | מוצרים, מחסנים, **תנועות מלאי** והתראות מלאי-נמוך |

| **דוחות ואנליטיקה** | **בונה דוחות על כל ערך במערכת** — עמודות ישות מובנות, שדות מותאמים, ותשובות *שאלון* (כולל עמודות טבלת "הוסף שורה" ו-multi-select, מפוצחות בזמן-אמת) — הכל ניתן לקיבוץ / סינון / צבירה, עם תוויות עברית לכל שדה. שמונה סוגי גרף + **Pivot דו-ממדי**; **פילוח זמן** (יום / שבוע / חודש / רבעון / שנה) עם **Δ% מול תקופה קודמת**; **סדרות מרובות** על גרף אחד; **drill-through** מכל אגרגט ישר לרשומות עצמן; תצוגה-חיה; ייצוא Excel / CSV; ו**כלי SQL לקריאה-בלבד** לאדמינים (SELECT בלבד, מבודד בטרנזקציה שתמיד מתגלגלת-אחורה, מוגבל שורות/זמן, מתועד במלואו). דוחות שמורים ניתנים ל**שליחה מתוזמנת במייל**, ל**התראות סף** ("שלח מייל אם לידים חדשים < 10 השבוע"), ל**שיתוף בלינק ציבורי / הטמעה ב-iframe**, ולסידור על **דאשבורד בגרירה** (סידור + שינוי-רוחב, נשמר אוטומטית). **הרשאות הצפייה מנוהלות פר-מחלקה**, ו**"שאל את הנתונים" ב-AI** הופך שאלה בשפה חופשית לדוח מוכן. לצד זה: דאשבורד KPI מנהלים **חי** (משפך, תפוסה, שיעור no-show, עומס-משימות) המתרענן על WebSockets, וידג'טים מוצמדים, ודוח פעילות-משתמש |
| **הודעות** | תבניות **SMS ומייל ניתנות-לעריכה** במערכת עם merge tokens, בשימוש מנוע האוטומציות (עורכים את הנוסח פעם אחת — כל הכללים מתעדכנים); **מיילים ממותגים ממוקדי-deliverability** (HTML RTL עם CSS מוטמע + חלופת טקסט-נקי + Reply-To להורדת ציוני-ספאם) |
| **צמיחה** | בונה **דפי-נחיתה** מתארחים → לידים, מחברי-פרסום, **webhooks יוצאים חתומים ב-HMAC** |
| **ניהול ואבטחה** | **עץ הרשאות גרנולרי** ויזואלי (מודול → טאב → שדה, allow/deny פר תפקיד / קבוצה / משתמש, deny-by-default לפעולות רגישות); ניהול משתמשים / תפקידים / קבוצות; **2FA אופציונלי** (TOTP + מייל OTP); **נעילת-סשן אוטומטית** בחוסר-פעילות; **יומן אירועי-התחברות** שמתעד את הסיבה האמיתית להצלחה/כישלון של כל התחברות (מסך ההתחברות נשאר גנרי כדי למנוע account enumeration); הגדרות תקשורת / טלפוניה / AI / שימור / ספריה ניתנות-לעריכה |
| **מערכת עיצוב ו-UX** | שכבת **tokens יחידה** (צבע / טיפוגרפיה / רדיוס / צל / תנועה) מניעה כל מסך ב**בהיר וכהה** — בלי ערכים קשיחים, נאכף ב-lint; ספריית פרימיטיבים; **RTL/LTR** מלא דרך תכונות-CSS לוגיות; ומסך-התחברות **glassmorphism** (aurora מונפש, פאנלים מעורפלים, מודע ל-reduced-motion) |

## 🛠️ טכנולוגיה והנדסה

- **צד-שרת:** ASP.NET Core Web API (**.NET 10 LTS**), EF Core, SQL Server, JWT + רוטציית refresh, מדיניות RBAC, **SignalR** (WebSockets) לדחיפה בזמן-אמת, ג'ובי **Hangfire** ברקע (תזכורות, הסלמה, שימור, שליחת דוחות מתוזמנת, ניקוד AI לילי).
- **צד-לקוח:** React 19 + TypeScript + Vite + MUI (RTL מלא) + react-i18next (החלפת he/en חיה), מעוצב ע"י **מערכת עיצוב מבוססת-tokens** (שכבת design-token יחידה המזינה סט-רכיבים מעוצב: רייל-ניווט כהה, צבעי-סטטוס סמנטיים, כרטיסים/טבלאות בסגנון enterprise), עם **חיפוש-קפיצה גלובלי** בטופ-בר וחיפוש-ניהול מודע-מילים-נרדפות בכל מסכי ההגדרות.
- **אינטגרציות:** מתאמים provider-agnostic לטלפוניה (CTI), AI, SMS / מייל / WhatsApp ומקורות-לידים — לכל אחד סימולטור "dev", כך שכל תהליך ניתן להדגמה ולבדיקה בלי חשבון צד-שלישי.
- **חוצי-מערכת:** audit אוניברסלי, בידוד רב-שוכרים, תור-ג'ובים עם retries, בדיקות-הרשאה deny-by-default עד רמת השדה.
- **בטיחות-נתונים בתכנון:** שום דבר אינו נמחק קשיח בשימוש רגיל — soft-delete גלובלי הופך כל מחיקה למצב-ניתן-לשחזור, המוצג ב**סל-מחזור לאדמין** (מי מחק מה, מתי, עם ערכי-השדות המלאים ושחזור בקליק). פעולות הרסניות משתמשות באישור type-to-confirm וב-**undo** מוטמע; ה"purge" האמיתי היחיד הוא פעולה ייעודית מגודרת-הרשאה המסירה רק שורות שכבר בסל.
- **הקשחת פרודקשן (On-Prem):** נארז כ-**שירות Windows** המגיש את ה-API *ואת* ה-SPA הבנוי מתהליך/פורט אחד; **לוגים מתגלגלים מובנים**; **טיפול-שגיאות גלובלי** (פירוט מלא ללוג, שגיאה גנרית נקייה ללקוח); endpoints של התחברות / OTP עם **rate-limit** (הגנה מפני brute-force); **health-check עמוק** המאמת קישוריות-DB; ו-**EF transient-fault retry** לעמידות מול שרת SQL נפרד. סקריפט publish בפקודה אחת + runbook פריסה מלא.
- **איכות:** GitHub Actions CI (build + tests לצד-שרת, build לצד-לקוח), מיגרציות EF, ו**חבילת בדיקות xUnit** גדֵלה (EF In-Memory) המכסה שלמות-חתימה וזיהוי-זיוף, חשבון-כסף בצד-שרת ושער אישור-ההנחה, webhooks טלפוניה (התאמת-מתקשר, idempotency, שער-סוד, ניתוב IVR), ניקוד NPS, חשבון תחזית-משוקללת והשגת-יעדים, מנוע-התובנות, חוזה מיזוג-הכפילויות, מנוע SLA לפניות, threading/נרמול של התיבה, ותאימות הסכמה/מעקב/הסרה בקמפיינים.
